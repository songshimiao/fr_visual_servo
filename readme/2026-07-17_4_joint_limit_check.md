# 关节限位检查功能说明

## 功能概述

新增了关节限位预检查功能，用于在下发机器人运动指令前主动检查目标关节位置是否超出安全范围，避免因关节超限导致的机械臂停止响应问题。

## 问题背景

在之前的系统中，Inverse Kinematics (IK) 求解器 `ikine_LM` 是纯数值迭代求解，不会主动将解约束在实际的关节限位之内。一旦将超限的解通过 ServoJ 下发给控制器，控制器会报 "PTP指令关节超限" 等错误，导致机械臂直接"挂掉"停止响应，必须通过 WebApp 手动复位才能继续，这严重影响了系统的可用性和安全性。

## 解决方案

新增了 `check_joint_limits` 函数和相应的检查逻辑：

### 1. 新增的检查函数
在 `FairinoRobot` 类中新增了 `check_joint_limits` 方法：
```python
def check_joint_limits(self, joint_pos: JointPos, margin_rad: float = 0.0):
    """
    检查给定关节位置是否超出(或过于接近)关节限位。
    
    :param joint_pos: 待检查的关节位置 (rad)
    :param margin_rad: 安全裕度 (rad)，正值表示比真实限位更早触发告警
    :return: (ok, violations)
             ok: bool, 全部关节都在(收紧后的)限位内时为 True
             violations: List[(joint_index, q_rad, qmin_rad, qmax_rad)]
    """
```

### 2. 实时检查机制
在主控制循环中增加了实时关节限位检查：
- 在每次 IK 求解后，检查目标关节位置是否超出安全范围
- 如果发现超限，系统会：
  1. 不下发本次 ServoJ 指令（机械臂保持不动）
  2. 不将超限解作为下一次 IK 的种子
  3. 在预览窗口红色高亮提示警告
  4. 节流打印日志信息
  5. 继续主循环，等待目标恢复到可达范围后自动恢复

### 3. 安全裕度参数
新增 `--joint-limit-margin-deg` 参数，允许用户自定义安全裕度（默认3°）：
- 当目标关节角距离真实限位小于安全裕度时，系统会提前预警
- 提供了更保守的保护机制，避免刚好卡在硬限位上

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--joint-limit-margin-deg` | 3.0 | 关节限位安全裕度(度)，提前预警的缓冲区大小 |

## 使用示例

```bash
# 启用关节限位检查功能
python scripts/main_pbvs.py --execute --joint-limit-margin-deg 5.0
```

## 优势

1. **预防性保护**：在问题发生前主动拦截，避免机械臂"挂掉"
2. **用户体验优化**：避免了复杂的复位操作，系统自动恢复
3. **安全性提升**：提供更可靠的安全保护机制
4. **灵活性**：支持自定义安全裕度，适应不同应用场景
